# Validation Notes

## Scope

This package archives the structured validation artifacts for the computation
of A224540 through `a(28)`.

The production program performs an iterative DFS on the inverse Collatz tree
rooted at 1 with bound `M = 3^n`. For each node `x`, the possible predecessors
are `2*x` and, when valid, `(x-1)/3`. The odd predecessor 1 of 4 is excluded by
the production rule `y > 1`, avoiding the trivial `1,4,2,1` cycle.

## Included Structured Artifacts

- `campaigns/a22_a25/a22_a25.tsv`
- `campaigns/a26/aggregate_n26.tsv`
- `campaigns/a27/aggregate_n27.tsv`
- `campaigns/a28/aggregate_n28.tsv`
- all segment TSVs needed to independently reconstruct the aggregate counts
- per-command manifests generated by the production binary
- `run_manifest.tsv`
- `A224540_new_results_independent_validation.md`

The manifest files preserve the original command lines used during the local
campaigns, including their original `runs/...` output paths. The corresponding
TSV and manifest artifacts copied into this publication package are stored under
`validation/campaigns/...`.

## Independent Artifact-Level Validation

Run:

```bash
python3 tools/a224540_validate_new_results.py
```

The validator independently parses TSV files, validates the extended b-file,
recomputes powers `3^n`, reconstructs prefix/frontier summaries for segmented
campaigns, checks complete segment coverage, and recomputes aggregate sums.

It does not call `build/a224540` and does not perform a second full traversal
of every subtree for `n = 28`.

## Runtime Imbalance

Segments are used for resumability and auditability, not for equal workload.
For `a(28)`, equal-size intervals of the depth frontier showed highly unequal
runtime because subtree sizes in the inverse Collatz tree are irregular.

This imbalance is part of the computational record and is preserved in the
segment TSV files.
